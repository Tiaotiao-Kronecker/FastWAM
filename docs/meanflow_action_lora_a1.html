1. 方案动机

旧的 C-action-only MeanFlow 放开了 full action expert 和 proprio encoder。它在 2k 到 effective 8k 的训练里没有解决 LIBERO-10 的 0/4 失败， trace 也显示 long-horizon gripper/action phase 被破坏。新的 A1 方案不再默认 full action dense finetune，而是使用受控容量扩展。

方案 判断 主要风险
conditioner-only 最稳，但可能只是 release time modulation 的小标定。 容量不足，无法表达真正的平均速度场。
full action expert 容量最大。 容易破坏 release policy semantics，旧 action-only 已显示 gripper/task timing 漂移。
A1 conditioner + head + LoRA 在 release body 上增加低秩、零初始化的可训练修正。 容量仍可能不足，但 failure 更容易定位。
A1 的目标不是证明 MeanFlow 一定只需小改动，而是避免一开始就把“平均速度 readout 适配”和“release policy 语义漂移”混在一起。

2. MeanFlow 与 Equal-Time 边界

动作流的训练路径记为：

\[ x_t = (1-t)a_0 + t\epsilon,\qquad v = \epsilon - a_0 \]

A1 学习区间平均速度：

\[ u_\theta(x_t,t,r\mid c),\qquad 0 \le r < t \le 1 \]

finite-difference MeanFlow 主目标沿用当前实现形式：

\[ u_{\mathrm{target}} = v - (t-r)\frac{\partial u_\theta}{\partial t} \]

新增 equal-time anchor 使用 MeanFlow 自身边界条件：

\[ u_\theta(x_t,t,t\mid c)=v(x_t,t\mid c) \]

这个 anchor 不等价于把所有 interval 拉回 flow matching。它只在 \(r=t\) 或 near-equal branch 上启用，用来防止 LoRA/head 把小 interval 的 release velocity semantics 搅乱。

3. Action-LoRA 数学与实现约束

对一个原始线性层 \(y=xW_0^\top+b\)，LoRA 增加低秩残差：

\[ y = xW_0^\top+b + \frac{\alpha}{r}xA^\top B^\top \]
符号 含义 A1 设置
\(W_0,b\) release checkpoint 中已有的 dense weight 和 bias。 冻结，不直接训练。
\(A\) down projection，形状约为 \(r\times d_{in}\)。 随机小初始化。
\(B\) up projection，形状约为 \(d_{out}\times r\)。 零初始化，保证 step0 等价 release。
\(\alpha/r\) LoRA scale。 第一版用 \(\alpha=r=4\)。

State Dict 兼容要求

A1 的 LoRA wrapper 必须保留原始 nn.Linear 的 weight 和 bias key。 这样 release checkpoint 的 self_attn.q.weight 等权重能正常加载，LoRA 只新增 lora_A 与 lora_B。 

class LoRALinear(nn.Linear):
    def __init__(self, base: nn.Linear, rank: int, alpha: float, dropout: float):
        super().__init__(
            in_features=base.in_features,
            out_features=base.out_features,
            bias=base.bias is not None,
            device=base.weight.device,
            dtype=base.weight.dtype,
        )
        self.weight.data.copy_(base.weight.data)
        if base.bias is not None:
            self.bias.data.copy_(base.bias.data)
        self.weight.requires_grad_(False)
        if self.bias is not None:
            self.bias.requires_grad_(False)

        self.lora_A = nn.Parameter(...)
        self.lora_B = nn.Parameter(torch.zeros(...))
        self.scaling = alpha / rank

    def forward(self, x):
        base = F.linear(x, self.weight, self.bias)
        delta = F.linear(F.linear(self.dropout(x), self.lora_A), self.lora_B)
        return base + self.scaling * delta

4. LoRA 代码结构归档

A1 的 LoRA 不是单独训练一个新 action expert，而是在 release action expert 的若干 nn.Linear 上安装低秩残差。核心约束是：base weight/bias 原样从 release checkpoint 加载并冻结，只有 lora_A/lora_B 参与 optimizer。

函数/模块 职责 关键行为
LoRALinear 包装单个 nn.Linear。 保留 weight/bias key，新增 lora_A/lora_B；lora_B 零初始化。
matches_lora_target(name, targets) 判断模块名是否属于 action LoRA target。 第一版只匹配 action expert 内部模块，避免误装到 video expert 或 shared wrapper。
install_action_lora(model, cfg) 遍历 action expert blocks 并替换目标 nn.Linear。 替换后立即冻结 base 参数，返回安装数量和 LoRA 参数量用于日志。
configure_trainable_parameters() 统一控制训练范围。 冻结 video/proprio/action base，打开 conditioner、action head、LoRA。 
def install_action_lora(action_model, targets, rank=4, alpha=4, dropout=0.0):
    installed = []
    for module_name, parent, child_name, child in iter_named_linears(action_model):
        if not matches_lora_target(module_name, targets):
            continue
        wrapped = LoRALinear(child, rank=rank, alpha=alpha, dropout=dropout)
        setattr(parent, child_name, wrapped)
        installed.append(module_name)
    return installed

def configure_trainable_parameters(self):
    self.requires_grad_(False)
    self.freeze_video_expert()
    self.proprio_encoder.requires_grad_(False)

    enable_train(self.meanflow_joint_time_embedding)
    enable_train(self.meanflow_joint_time_projection)
    enable_train(self.action_head)
    enable_lora_only(self.action_model)
这里记录的是 A1 代码改动的目标形态。落地实现时需要先做 zero-update parity：安装 LoRA 后不训练， 使用同一 release checkpoint 和同一 eval config，成功率和 action trace 应接近 release_1。

5. 与 MoT 的兼容性

MoT 没有独立的一套 shared q/k/v 权重。它在每层分别调用 video expert 和 action expert 的 block，得到各自的 q/k/v， 再把 token 维度 concat 后做 mixed attention。因此 Action-LoRA 只改变 action expert 的投影输出，不改变 MoT 的接口。

路径 A1 行为 影响
video expert q/k/v 冻结，不装 LoRA。 video cache 与 release 保持一致。
action expert q/v/o、cross-attn、FFN 装 LoRA，base weight 冻结。 给 action branch 增加低秩 x-dependent 修正能力。
MoT mixed attention 无需改动。 输入输出 tensor shape 不变。
评测 A1 checkpoint 时必须使用包含 LoRA wrapper 的 A1 config。否则 checkpoint 中的 LoRA key 会被忽略，评出来可能不是实际训练后的模型。

6. Loss 组成

L_total =
    lambda_meanflow_target * L_meanflow_target
  + lambda_equal_time_velocity * L_equal_time_velocity
Loss 定义 A1 默认 备注
L_meanflow_target finite-difference MeanFlow target MSE。 1.0 主训练目标。
L_equal_time_velocity 在 r=t branch 上对齐 \(u_\theta(x_t,t,t)\) 与 \(v=\epsilon-a_0\)。 0.1 MeanFlow 边界条件，不是 release teacher。
L_action_velocity 旧辅助项，对所有 sampled interval 对齐瞬时 velocity。 0.0 不要在 A1 主线打开，否则会把大 interval 拉回普通 flow matching。
L_action_endpoint 旧 endpoint 辅助项。 0.0 随机 \(r>0\) 时 target 不应直接是 \(a_0\)，第一版不启用。

model:
  one_step_meanflow:
    objective: finite_difference
    random_timesteps: true
    equal_time_prob: 0.0
    trainable_scope: conditioner_head_lora
    train_proprio_encoder: false
    conditioner_mode: joint_delta
    freeze_video_expert: true
    derivative_epsilon: 0.05

    equal_time_anchor_prob: 0.25

    lora_rank: 4
    lora_alpha: 4
    lora_dropout: 0.0
    lora_layers: all

      - self_attn.q
      - self_attn.v
      - self_attn.o
      - cross_attn.q
      - cross_attn.o
