FastWAM One-Step 原理与实现对照

统一符号

FastWAM 的 action scheduler 使用 sigma convention： sigma=0 是真实 action，sigma=1 是纯噪声。为了避免和不同论文中的 t 方向混淆，本文主公式都使用 sigma。

图 1：本文使用 FastWAM 代码里的 sigma 方向。采样时 sigma 从 1 降到 0。

基础路径

a        : 真实 action 序列，来自数据集并经过 dataset stats normalization
epsilon  : 标准高斯噪声
sigma    : 噪声强度，sigma in [0, 1]
x_sigma  = (1 - sigma) * a + sigma * epsilon
v*       = epsilon - a

因为 x_sigma 对 sigma 的导数就是 epsilon - a， 所以 flow matching 的目标速度是 v* = epsilon - a。

采样更新

给定当前状态 x_sigma 和模型预测速度 v_theta(x_sigma, sigma):

d = sigma_current - sigma_next, 其中 d > 0
x_next = x_sigma - d * v_theta(x_sigma, sigma)

若 num_inference_steps = 1:
  sigma_current = 1, sigma_next = 0, d = 1
  x_0 = x_1 - v_theta(x_1, 1)
重要：Shortcut 论文常用 x_t = (1-t) x_0 + t x_1，其中 x_0 是噪声，x_1 是数据，采样方向是 t 增大。 FastWAM 代码里 sigma=1 是噪声、sigma=0 是数据，采样方向是 sigma 减小。两者本质一致，但符号方向相反。

论文方法、训练数据、Loss 与代码实现总表

这一节把后面几种 one-step 方法先压缩成一张判断表。这里的“训练数据制作”不是指重新采集数据， 而是指从现有 batch 的真实 action、状态、图像和语言条件中如何构造 noisy action、timestep、step size 以及监督 target。

方法 论文/理论目标 训练数据制作 Loss 是否和论文一致 FastWAM 代码落点
Release baseline 普通 flow matching：学习瞬时速度 v*=epsilon-a 用真实 action a 与高斯噪声 epsilon 构造 x_sigma=(1-sigma)a+sigma epsilon，随机 sigma 一致。这就是 FastWAM 原始 action flow loss fastwam.py::training_loss，scheduler_continuous.py
Endpoint one-step 无独立论文；工程 baseline：固定最大噪声直接回 action 固定 sigma=1，输入纯噪声 action，target 是 epsilon-a 和端点 a 不对应论文方法；且 velocity loss 与 endpoint loss 在 sigma=1 时数学等价 fastwam_one_step_action.py

Shortcut official-style Shortcut Models：step-conditioned flow，逐级 bootstrap 到大步 batch 内随机混合 flow 样本和 bootstrap 样本；使用 dt_base=-log2(d) 较一致。仍是迁移到 action-only FastWAM fine-tune，训练规模与论文不同 fastwam_one_step_shortcut_official.py

当前结论：如果继续投入 one-step 方法，优先级应是 Shortcut official-style 的更长训练和更合适 eval 集；MeanFlow 需要先改成论文 JVP 形式， 否则现有 collapse 不能被解释成 MeanFlow 方法本身失败。

Release Baseline：官方 FastWAM 多步/一步采样

这是最重要的对照组。release_1、release_4、release_10 不是不同训练版本， 而是同一个官方 checkpoint 在 eval 时传不同的 EVALUATION.num_inference_steps。

理论公式：普通 Flow Matching

Flow Matching 的基础思想来自 Lipman et al. 2022。在线性路径上，训练模型预测 x_sigma 沿噪声强度变化的速度。 

x_sigma = (1 - sigma) * a + sigma * epsilon
v*      = d x_sigma / d sigma = epsilon - a

L_action = E[ w(sigma) * || v_theta(x_sigma, sigma, obs, instruction) - v* ||^2_masked ]

L_total = lambda_video * L_video + lambda_action * L_action

代码实现对照

理论步骤 本仓库实现
采样 action 噪声 epsilon 和训练 timestep/sigma src/fastwam/models/wan22/fastwam.py 的 training_loss 中 noise_action、sample_training_t
构造 x_sigma scheduler_continuous.py 的 add_noise：(1-sigma)*sample + sigma*noise
目标速度 epsilon - a training_target(...) 返回 noise - sample
masked MSE action padding 通过 action_is_pad 排除，避免 padding token 参与 loss
推理 Euler step step(model_output, delta, sample)；因为 delta = sigma_next - sigma_current 为负，所以等价于 x - d*v
我们现在的 baseline 定义：官方 release checkpoint 直接下载后即可推理； 需要配套 *_dataset_stats.json 做 action/state normalization。除了 shortcut/meanflow 等 one-step 方案， baseline 本身没有再 fine-tune。

Endpoint One-Step：直接端点微调

这是我们最早尝试的一步化方案。它没有对应独立论文方法，更像一个直接工程 baseline： 固定从 sigma=1 的纯噪声出发，让模型一步回到真实 action。

公式

固定 sigma = 1:
x_1 = epsilon
v*  = epsilon - a

模型预测:
v_theta = f_theta(x_1, sigma=1)
a_hat   = x_1 - 1 * v_theta

L_velocity = || v_theta - v* ||^2_masked
L_endpoint = || a_hat - a ||^2_masked
L_endpoint_one_step = 0.5 * L_velocity + 0.5 * L_endpoint

一个关键等价关系

当 sigma=1 时，endpoint loss 和 velocity loss 本质上是同一个误差： 

a_hat - a
= (epsilon - v_theta) - a
= (epsilon - a) - v_theta
= v* - v_theta

所以 ||a_hat - a||^2 = ||v_theta - v*||^2

这解释了为什么 endpoint one-step 看起来有两个 loss，但并没有提供两个独立监督信号。

训练数据制作

Endpoint one-step 不改变原始 demonstration 数据。每个 batch 仍来自 LIBERO/RoboTwin 的真实观测、语言和 action chunk； 变化只发生在 action latent/token 侧：丢弃随机中间 sigma，固定用最大噪声状态训练。 

给定 batch:
  image/context/proprio/instruction/action = data loader 输出
  a = normalized action chunk
  epsilon ~ N(0, I)

构造 one-step 样本:
  sigma = 1
  x_1 = epsilon
  target_velocity = epsilon - a
  target_endpoint = a

Loss 是否和论文一致

这个方法没有对应的 one-step 论文公式；它是我们为 FastWAM 写的工程 baseline。 因为 L_endpoint 和 L_velocity 在 sigma=1 时等价， 所以它更像“重复加权的最大噪声 flow matching”，不是 Shortcut/MeanFlow 那类有额外理论约束的 loss。

代码实现对照

公式项 实现位置
sigma=1 或最大 timestep fastwam_one_step_action.py 的 _build_one_step_action_timestep
构造 noisy_action 与 target_action_velocity training_loss 中 add_noise(action, noise_action, timestep_action) 和 training_target(...)
a_hat = noisy_action - sigma * pred_velocity pred_action_endpoint = noisy_action - sigma * pred_action_velocity
L_velocity loss_action_velocity
L_endpoint loss_action_endpoint
判断：endpoint 微调适合作为 sanity check，但它不是很强的一步化理论。 它只告诉模型“从纯噪声直接到数据”，没有教模型“大步”和“多个小步”之间如何一致。

Shortcut Models：一步/多步统一的 step-size conditioned flow

Shortcut Models 的出处是 Kevin Frans, Danijar Hafner, Sergey Levine, Pieter Abbeel 的 One Step Diffusion via Shortcut Models。核心改动是：模型不仅看当前噪声水平， 还看这一步想走多远，也就是 step size d。

论文公式：step-conditioned velocity

用论文方向写，t 从噪声走向数据：

x_t = (1 - t) * x_0 + t * x_1
s_theta(x_t, t, d) ≈ (x_{t+d} - x_t) / d
x_{t+d} = x_t + d * s_theta(x_t, t, d)

用 FastWAM sigma 方向重写：

x_sigma = (1 - sigma) * a + sigma * epsilon
u_theta(x_sigma, sigma, d) ≈ (x_sigma - x_{sigma-d}) / d
x_{sigma-d} = x_sigma - d * u_theta(x_sigma, sigma, d)

论文公式：两个半步要等于一个大步

论文方向：

s_theta(x_t, t, 2d)
≈ 0.5 * [
    s_theta(x_t,       t,     d)
  + s_theta(x_{t+d},   t+d,   d)
  ]

FastWAM sigma 方向：

u_theta(x_sigma, sigma, 2d)
≈ 0.5 * [
    u_theta(x_sigma,       sigma,     d)
  + u_theta(x_{sigma-d},   sigma-d,   d)
  ]
图 2：Shortcut 的训练信号不是只看端点，而是约束“一步大跳”和“两步小跳”的平均速度一致。

训练数据制作：论文视角

Shortcut 不需要额外标注数据。它从同一个真实样本 (obs, instruction, a) 出发， 额外采样噪声 epsilon、噪声位置 sigma 和步长 d，把一个普通 action chunk 变成“某个噪声位置、某个步长下应该走的速度”训练样本。 

Flow grounding 样本:
  a, obs, instruction 来自数据集
  epsilon ~ N(0, I)
  选择最小步长 d_min = 1 / N
  选择 sigma 所在 section
  x_sigma = (1 - sigma) * a + sigma * epsilon
  target = epsilon - a

Bootstrap 样本:
  选择较大步长 d = 2^(-dt_base)
  x_sigma = (1 - sigma) * a + sigma * epsilon
  v1 = stopgrad(u_theta(x_sigma, sigma, d/2))
  x_mid = x_sigma - (d/2) * v1
  v2 = stopgrad(u_theta(x_mid, sigma-d/2, d/2))
  target = 0.5 * (v1 + v2)

论文训练：flow grounding + bootstrap

论文官方代码把 batch 混成两类样本：

样本类型 目标 作用
Flow matching 小步样本 v_t = x_1 - x_0，对应最小 step size 给模型一个真实、稳定的小步速度锚点
Bootstrap 大步样本 v_target = 0.5 * (v_half_1 + v_half_2) 用两个更小 step 的模型预测蒸馏出一个更大 step 的目标

我们旧版 shortcut 实现

旧版对应 fastwam_one_step_shortcut.py。 它加入了 step-size conditioner，但训练分布是固定的：默认 start_timestep=null 代表最大 timestep， step_size=1.0 代表一步从噪声到 action。 

固定:
sigma_start = 1
d = 1

pred_large = u_theta(x_sigma, sigma, d)
pred_half1 = u_theta(x_sigma, sigma, d/2)
x_mid      = x_sigma - (d/2) * stopgrad(pred_half1)
pred_half2 = u_theta(x_mid, sigma-d/2, d/2)

target_v   = epsilon - a
target_big = 0.5 * (stopgrad(pred_half1) + stopgrad(pred_half2))

L_old_shortcut =
  lambda_v        * MSE(pred_large, target_v)
+ lambda_endpoint * MSE(x_sigma - d*pred_large, x_sigma - d*target_v)
+ lambda_cons     * MSE(pred_large, target_big)
+ lambda_half     * 0.5 * [MSE(pred_half1, target_v) + MSE(pred_half2, target_v)]
旧版的主要偏差：它固定在 sigma=1,d=1，没有像论文一样从小步 flow target 逐级 bootstrap 到大步；step conditioning 也不是论文代码里的 dt_base=-log2(d) 离散编码。 早期训练还存在额外 conditioner 参数未进入 optimizer 的问题，后来已通过 extra_trainable_parameters() 修正。

我们修正版 official-style shortcut

修正版对应 fastwam_one_step_shortcut_official.py， 目的是更接近 Shortcut Models 官方 JAX 代码。

数据制作上，它仍然使用原 FastWAM 的 action chunk、图像 context、proprio 和语言 instruction； 新增的是 per-sample 的 shortcut_dt_base 条件和 flow/bootstrap target。代码里用 shortcut_bootstrap_ratio 决定当前 batch 走 grounding 还是 bootstrap。 

配置:
num_denoise_steps = 128
max_dt_base = log2(128) = 7
d = 2^(-dt_base)

Flow grounding:
  随机 section_index
  sigma = 1 - section_index / 128
  dt_base = 7  # 最小步长 d=1/128
  target = epsilon - a

Bootstrap:
  随机 dt_base in {0, ..., 6}
  d = 2^(-dt_base)
  half_dt_base = dt_base + 1
  half_step = d / 2

  v1 = stopgrad(u_theta(x_sigma, sigma, half_dt_base))
  x_mid = x_sigma - half_step * v1
  v2 = stopgrad(u_theta(x_mid, sigma-half_step, half_dt_base))
  target = 0.5 * (v1 + v2)

Loss:
  L_shortcut_official = MSE(u_theta(x_sigma, sigma, dt_base), target)

Loss 是否和论文一致

official-style 版本的 loss 形式已经接近 Shortcut 论文：训练时只优化一个 target MSE， target 要么来自真实 flow grounding，要么来自两个半步 bootstrap。差异主要在任务域和训练制度： 论文是在生成模型训练流程里学习完整 shortcut flow；我们是在 FastWAM release checkpoint 上做 action-only fine-tune， video branch 基本冻结，训练步数也短得多。

论文/官方代码点 修正版实现
dt_base 条件，而不是直接给连续 step size _apply_shortcut_dt_base_conditioning，并用 sinusoidal embedding 注入 action expert 的 t_mod
最小步长 flow matching target _sample_flow_shortcut_batch，shortcut_dt_base=max_dt_base，target 为 noise-action
大步由两个半步 bootstrap _sample_bootstrap_shortcut_batch + with torch.no_grad() 计算 v1,v2
单一 shortcut target loss loss_shortcut_target，配置里 lambda_shortcut_target=1.0

第一性原理分析：为什么 shortcut 可能表现差

  • 如果 release_1 已经很强，提升空间天然小。LIBERO 上 release_1 已经 95.80%，大部分错误不是单纯积分步数造成的。
  • 旧版 shortcut 的训练分布太窄。只在 sigma=1,d=1 训练，模型没有学会不同噪声水平和不同步长的一致关系。
  • 大步目标来自模型自身，容易把偏差递归放大。Bootstrap 的前提是半步模型已经可靠；如果半步不可靠，大步 target 就是错误平均。
  • 机器人 action 是多模态且受状态强约束。图像生成中的一步样本质量不等于控制成功率；一点 action 偏差可能导致任务失败。
  • 训练步数可能远不足。Shortcut 论文是从完整训练流程中学 step-conditioned flow；我们是对强 pretrained checkpoint 做短程 fine-tune。
  • conditioner 参数必须真的被优化。之前 trainer 只收集 model.dit 参数，新增的 conditioner 不会更新；现在已修复。

MeanFlow：直接学习区间平均速度

MeanFlow 出处是 Zhengyang Geng, Mingyang Deng, Xingjian Bai, J. Zico Kolter, Kaiming He 的 Mean Flows for One-step Generative Modeling。它的关键思想是： 普通 flow matching 学的是某个时间点的瞬时速度 v；一步生成真正需要的是从噪声到数据 整个区间的平均速度 u。

论文公式：平均速度

MeanFlow 论文常用 t 表示噪声强度，r < t 是目标区间的另一端：

z_t = (1 - t) * x + t * epsilon
v   = epsilon - x

u(z_t, r, t) = (z_t - z_r) / (t - r)
             = average velocity over interval [r, t]

one-step sampling:
  x_hat = epsilon - u(epsilon, r=0, t=1)

训练数据制作：论文视角

MeanFlow 同样不需要额外人工标注。它把同一个 action chunk 变成区间监督样本： 随机选择区间端点 r,t，构造 z_t，模型输入同时包含 r 和 t。 当 r=t 时，平均速度退化为普通瞬时速度；当 r=0,t=1 时，它就是完整 one-step 需要的平均速度。 

给定真实 action a 和噪声 epsilon:
  随机采样 0 <= r <= t <= 1
  z_t = (1 - t) * a + t * epsilon
  v = epsilon - a
  模型预测 u_theta(z_t, r, t)
  用 JVP 构造 u_target = v - (t-r) * du/dt

论文公式：MeanFlow identity

MeanFlow 推导出一个可训练恒等式，把平均速度 u 和瞬时速度 v 联系起来： 

u_target = v - (t - r) * d u / d t
L_meanflow = || u_theta(z_t, r, t) - stopgrad(u_target) ||^2

官方 JAX 实现用 jax.jvp 同时得到 u 和方向导数 du/dt： 

u, du_dt = jax.jvp(
  u_fn,
  (z_t, t, r),
  (v_g, 1, 0)
)
u_tgt = v_g - (t-r) * du_dt
图 3：MeanFlow 直接学习区间平均速度，而不是只学习每个时间点的局部切线。

我们当前 meanflow 实现

当前实现位于 fastwam_one_step_meanflow.py。 它把 MeanFlow 接到 FastWAM action 分支上，并采用有限差分近似 du/dt。

数据制作上，当前版没有随机采样 r,t，而是默认固定完整区间： r=0，t=1。这让训练目标集中在纯 one-step 端点，但失去了论文中用中间区间稳定学习平均速度的机制。 

默认配置:
r = meanflow_start_timestep = 0
t = meanflow_end_timestep = max timestep = 1
epsilon_fd = 0.05

u_t = u_theta(x_t, r, t)
x_{t-eps} = x_t - eps * v*
u_prev = u_theta(x_{t-eps}, r, t-eps)

du_dt ≈ (stopgrad(u_t) - stopgrad(u_prev)) / eps
u_target = v* - (t-r) * du_dt

a_hat = x_t - (t-r) * u_t

L_current_meanflow =
  0.5  * MSE(u_t, u_target)
+ 0.25 * MSE(u_t, v*)
+ 0.25 * MSE(a_hat, a)

Loss 是否和论文一致

当前 loss 不等同于论文 MeanFlow。论文的关键是通过自动微分 JVP 得到方向导数，并用 u_target = v - (t-r) du/dt 训练平均速度；我们现在用两个 forward 的有限差分近似， 且额外加入 velocity 和 endpoint loss。因此当前实验结果只能说明这个 FastWAM 适配版本失败， 不能说明严格 MeanFlow 方法失败。

与论文 MeanFlow 的关键差异

论文/官方实现 我们当前实现 风险
随机采样 r,t，其中一部分样本令 r=t 保留普通 velocity 监督 默认固定 r=0,t=1 训练只覆盖全区间，缺少中间区间的稳定约束
用自动微分 JVP 计算 du/dt 用有限差分近似，并且对两个预测都 stop-gradient 目标可能噪声大，且不等价于论文的方向导数训练
MeanFlow 从头训练，不依赖 teacher/distillation 从 FastWAM release checkpoint 短程 fine-tune pretrained instantaneous flow 未必能平滑转成 average flow 参数化
官方 loss 主要是 normalized meanflow target loss 额外加 velocity/endpoint loss 三个目标可能相互拉扯，尤其 u 不应该总等于瞬时 v
当前实验解释：LIBERO 上 meanflow_1_2k 只有 6.70%。 这不是轻微调参问题，更像目标函数/实现形式与论文 MeanFlow 不匹配。 若继续做 MeanFlow，应优先改成 JVP 版本、随机 r,t 采样，并确认是否从头训练或更长程训练。

实验启发：LIBERO 的 release_1 已接近饱和，one-step 方法很难显示提升； RoboTwin quick 上 release_1 和 release_4/10 有更大差距，因此更适合做 one-step 方法是否有效的初筛。

最容易出错的地方

1把 release_1/4/10 当成不同权重

它们是同一个 official checkpoint，只是 eval 时 num_inference_steps 不同。

2漏掉 dataset stats

action/state 需要使用对应的 *_dataset_stats.json 归一化和反归一化；否则模型输入输出尺度会错。

3符号方向混淆

Shortcut 论文的 t 方向和 FastWAM 的 sigma 方向相反；更新符号也相反。

4只固定 sigma=1 训练

一步 endpoint 或旧 shortcut 没有覆盖中间噪声水平，难以学到稳定 step-conditioned flow。

5新增 conditioner 没进 optimizer

新增模块如果挂在 action_expert 下，trainer 必须显式收集；现在通过 extra_trainable_parameters() 处理。

6MeanFlow JVP 被有限差分替代

当前实现不是论文严格形式；finite difference target 很可能是 collapse 的主因之一。

7短训练不能代表论文方法

Shortcut/MeanFlow 论文都不是“在强 release checkpoint 上训 2k steps”这么简单，短 pilot 只能判断当前实现是否有希望。

8quick subset 不能替代 full eval

RoboTwin quick clean20 适合方向筛选；最终结论仍要扩大任务数和 episodes。

建议的下一步判断标准

  1. 任何新 one-step 方法都先和同一 checkpoint 的 release_1 比，而不是只和 release_10 比。
  2. LIBERO full average 因 release_1 已接近饱和，更适合验证“不退化”；要看 one-step 空间，优先用 libero_long_horizon_v1 或官方 libero_long_official。
  3. RoboTwin quick 更适合初筛“有没有提升空间”，但最终仍要扩大任务数和 episodes。
  4. MeanFlow 若继续，应先修成论文 JVP + 随机 r,t，再做训练；否则当前 collapse 结果缺少可解释性。
  5. Shortcut 若继续，应优先用 official-style 目标，并在长程/困难 eval 集上同时报告 release_1、method_1、release_10。
