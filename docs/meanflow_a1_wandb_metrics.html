FastWAM MeanFlow A1 WandB 指标说明

解释 A1 Action-LoRA MeanFlow 训练中 WandB 配置项、训练曲线、参数/梯度分组、动作曲线和评估 artifact 的含义。

1. WandB 配置项

wandb:
  enabled: true
  workspace: null
  project: fast-wam
  group: meanflow-a1
  name: a1-lora4-jointdelta-eqanchor0.1-70k
  mode: online
字段 含义 建议
enabled 是否启用 WandB。 A1 主实验启用；smoke 可关闭。
workspace WandB entity。为 null 时使用本地默认账号。 多人协作时填团队 workspace。
project WandB project 名。 固定为 fast-wam。
group 实验组，用于聚合同一系列 ablation。 A1 使用 meanflow-a1。
name 单次 run 名。 包含 rank、conditioner、anchor、steps 等关键信息。
mode online 直接上传，offline 本地缓存。 集群网络不稳时用 offline。

2. A1 Run Config 参数含义

这些字段应该随 run 一起写入 WandB config，方便之后按 conditioner、LoRA rank、anchor 强度和数据规模筛选实验。

WandB config key 含义 A1 主线建议
model.one_step_meanflow.objective MeanFlow target 的实现方式。 finite_difference，保持与现有 C/D 实现兼容。
model.one_step_meanflow.conditioner_mode interval conditioner 输入形式。 joint_delta，同时给模型 \(t\) 和 \(t-r\)。
model.one_step_meanflow.trainable_scope 允许训练的模块集合。 conditioner_head_lora，防止误开 full action body。
model.one_step_meanflow.freeze_video_expert 是否冻结 video expert。 true，保持视觉 token 和 video cache 语义不变。
model.one_step_meanflow.train_proprio_encoder 是否训练 proprio encoder。 false，避免状态编码漂移影响 release policy。
model.one_step_meanflow.derivative_epsilon finite-difference 估计 \(\partial u/\partial t\) 的扰动大小。 0.05 起步；过小噪声大，过大近似偏。
model.one_step_meanflow.equal_time_anchor_prob 执行 \(r=t\) velocity anchor branch 的概率。 0.25 起步，用来守住小 interval 边界。
model.one_step_meanflow.lora_rank LoRA 低秩维度 \(r\)。 4；作为容量和稳定性的第一折中点。
model.one_step_meanflow.lora_alpha LoRA scale 中的 \(\alpha\)。实际 scale 为 \(\alpha/r\)。 4，让初始有效 scale 约为 1。
model.one_step_meanflow.lora_target_modules 安装 LoRA 的 action expert 线性层列表。 优先 q/v/o、cross-attn q/o 和 FFN output；不装 video expert。
loss.lambda_equal_time_velocity equal-time anchor 的 loss 权重。 0.1 起步；太大可能把大 interval 拉回 flow matching。
max_steps optimizer step 数。 4 GPU batch=1 主实验约 70000；8 GPU 约 35000。

3. Loss 曲线

WandB key 含义 健康趋势 异常信号
train/loss 总 loss，包含 MeanFlow 主目标和 equal-time anchor。 平稳下降，无尖峰爆炸。 下降但 rollout 失败，说明 loss 与闭环行为脱钩。
train/loss_meanflow_target finite-difference MeanFlow target MSE。 主下降曲线。 长期不降可能容量不足或 LR 太低。
train/loss_equal_time_velocity r=t branch 上 \(u_\theta\) 对齐瞬时 flow velocity 的 loss。 不应长期高于主 loss 太多。 高企说明 LoRA/head 破坏了小 interval velocity。
train/loss_action_velocity 旧全 interval velocity auxiliary。 A1 中应为 0 或不存在。 非零表示配置误开，可能把 MeanFlow 拉回普通 flow matching。
train/loss_action_endpoint 旧 endpoint auxiliary。 A1 中应为 0 或不存在。 随机 \(r>0\) 时误用可能引入错误监督。

4. MeanFlow 变量诊断

WandB key 含义 用途
train/meanflow_sigma_start batch 中 sampled \(r\) 的均值。 确认 \(r\) 分布正常。
train/meanflow_sigma_end batch 中 sampled \(t\) 的均值。 确认 \(t\) 分布正常。
train/meanflow_interval \(t-r\) 的均值。 如果长期过小，训练会偏向普通 flow matching；如果过大，finite-difference target 更难。
train/meanflow_dudt_rms finite-difference 估计的 \(\partial u/\partial t\) RMS。 过大可能表示 target 不稳定或 LoRA 更新过猛。
train/pred_mean_velocity_rms 模型预测平均速度 RMS。 用于观察 action scale 是否漂移。
train/target_meanflow_rms MeanFlow target RMS。 与 pred RMS 对比判断 scale mismatch。
train/equal_time_anchor_applied 当前 log 窗口是否执行了 equal-time anchor branch。 确认 equal_time_anchor_prob 生效。

5. 参数与梯度分组

WandB key 含义 判断
params/trainable_total 总可训练参数量。 用于确认没有误放开 full action body。
params/trainable_lora LoRA 参数量。 rank/layer 配置是否符合预期。
params/trainable_conditioner joint-delta conditioner 参数量。 应包含 meanflow_joint_time_embedding 和 meanflow_joint_time_projection。
params/trainable_head action head 参数量。 确认 head 解冻。
grad_norm/lora LoRA 参数梯度范数。 过大说明 adapter 正在强行改 release policy；长期为 0 说明 LoRA 未参与训练。
grad_norm/conditioner conditioner 梯度范数。 判断 interval conditioner 是否承担主要学习。
grad_norm/head action head 梯度范数。 过大可能导致 action scale 漂移。
train/grad_norm 全局梯度范数。 用于发现整体数值不稳定。

6. 数据、学习率与性能指标

WandB key 含义 用途
train/lr 当前学习率。 确认 warmup/cosine schedule 正常。
performance/steps_per_sec optimizer steps per second。 估算 30k/70k run 的耗时。
performance/samples_per_sec 全局 samples per second。 等于 steps/s × batch × world size。
data/pad_retry_rate 启用 skip_padding_as_possible 后替换 padding window 的比例。 高比例表示 episode 边界/padding 影响较大。
data/action_pad_fraction batch 中 action padding token 比例。 A1 期望尽量低。
data/* 指标需要在 dataset 或 collate 侧补充返回统计；第一版至少应通过离线数据 audit 确认 padding 比例。

7. 可视化 Artifact

Artifact 内容 用途
eval/video 已有 eval MP4，展示 pred / VAE recon / GT 拼接。 发现视觉路径或动作 rollout 明显异常。
eval/action_curve_pred_vs_gt 固定 diagnostic batch 上 7 维 action 曲线：xyz、rot、gripper。 观察 gripper timing 和 action phase 是否漂移。
eval/action_curve_endpoint 从 \(t=1,r=0\) one-step endpoint 还原出的动作曲线。 判断推理路径是否与训练 loss 对齐。
eval/trace_summary_table action trace summary：first close、close ratio、transition count、episode length。 rollout 成败背后的动作相位诊断。

8. 常见异常模式

现象 可能原因 处理
loss_meanflow_target 下降，但 rollout 不涨。 MeanFlow target 与闭环 action phase 脱钩。 看 action curve 与 trace summary；考虑加强 equal-time 或加入 r=0 endpoint ablation。
loss_equal_time_velocity 高或震荡。 LoRA/head 破坏小 interval release velocity。 降低 LR、降低 LoRA rank、增加 anchor probability。
grad_norm/lora 远大于其他组。 LoRA 在强行重写 action policy。 降低 LoRA LR 或 rank；必要时改 last-layer/head-only ablation。
gripper transition 过多或长期不 close。 任务相位和 gripper timing 漂移。 优先看 action trace，不要只看训练 loss。
zero-update eval 低于 release_1。 LoRA wrapper 或 joint-delta 初始化破坏 release。 停训，检查 checkpoint load 和 zero-init。
