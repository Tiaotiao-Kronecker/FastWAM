归档 TwinFlow / SplitMeanFlow / MeanFlowSE 讨论，并制定 A2：interval mixture + clipped finite-difference MeanFlow + action-only LoRA 的训练计划。

状态：设计计划归档；尚未进入代码实现。

1. 结论摘要

A2 不应作为一次“大杂烩”实验直接改动 objective、interval sampling、capacity、data weighting 和 endpoint loss。 设计上应拆成可归因的 ablation ladder：先验证 interval mixture 和 residual clipping， 再考虑 endpoint、困难任务上采样和 rank 8 扩容。
设计点 A2 采用方式 原因
归因原则 A2-core 保持 A1 架构、数据、LoRA rank 和主要超参不变；只逐步改变 interval sampling 与 residual target 稳定项。 如果性能变化，能知道来自 sampling、clipping、endpoint、数据权重还是容量。
Action-only 范围 继续冻结 video 和 proprio；训练 interval conditioner、action head、action LoRA/adapter。 避免破坏 release policy 的视觉和 proprio 对齐，同时给 action expert 足够容量表达平均速度场。
Interval mixture 作为单独 ablation：A2-mix；随后与 clipping 组合成 A2-core。 让训练同时覆盖部署 one-step、flow matching 局部锚点和一般 MeanFlow 区间。
Residual target 用 TwinFlow 风格的积分残差替代显式 JVP。 避免 action model 上昂贵且脆弱的 JVP，同时保留 MeanFlow 固定点。
Residual clipping 作为单独 ablation：A2-clip；随后与 interval mixture 组合成 A2-core。 控制自举 target 的噪声，防止 finite-difference 残差把模型拉飞。
fake/adv branch 暂不采用 它是 TwinFlow 的分布匹配扩展，不是标准 MeanFlow；机器人动作 fake 分布的语义风险更高。

2. A1 结果与问题定位

A1 使用 release checkpoint 初始化，冻结 video/proprio，训练 joint-delta interval conditioner、action head 和少量 action LoRA，并加入 equal-time velocity anchor。

项 A1 当前设定 观察

数据 full LIBERO no-noops four-suite 数据。 不是早期小数据 finetune 的问题。

3. MeanFlow 的核心约束

记动作插值路径为：

\[ x_t = (1-t)a_0 + t\epsilon,\qquad v_t = \epsilon - a_0 \]

MeanFlow 模型输出从 \(t\) 到 \(r\) 的平均速度：

\[ u_\theta(x_t,r,t)\approx \frac{1}{t-r}\int_r^t v_s\,ds \]

总积分量为：

\[ I_\theta(t,r)=(t-r)u_\theta(x_t,r,t) \]

标准 MeanFlow 的 JVP 形式来自：

\[ \frac{d}{dt}I_\theta(t,r)=v_t \] \[ u_\theta + (t-r)\frac{d u_\theta}{dt}=v_t \] \[ u_{\mathrm{target}}=v_t-(t-r)\frac{d u_\theta}{dt} \]

官方 MeanFlow 和许多复现会直接用 JVP 估计 \(d u_\theta/dt\)。FastWAM 之前的 JVP pilot 也沿这条路尝试过，但在动作模型上代价高、实现脆弱、训练信号噪声较大。

4. TwinFlow 如何对上 MeanFlow

TwinFlow 可以理解为把上面的 JVP 目标改写成无显式 JVP 的 finite-difference residual。令模型输出：

\[ F_\theta(x_t,t,tt)\approx u_\theta(x_t,tt,t) \]

总积分量为：

\[ I_t=(t-tt)F_\theta(x_t,t,tt) \]

后退一个小步：

\[ x_{t-\Delta t}=x_t-\Delta t\,v_t \] \[ I_{t-\Delta t}=(t-\Delta t-tt)F_\theta(x_{t-\Delta t},t-\Delta t,tt) \]

有限差分残差为：

\[ R=\frac{I_t-I_{t-\Delta t}}{\Delta t}-v_t \]

展开可得：

\[ R\approx F_\theta+(t-tt)\frac{dF_\theta}{dt}-v_t \] \[ R\approx F_\theta-\left(v_t-(t-tt)\frac{dF_\theta}{dt}\right) \]

因此：

\[ F_\theta-R\approx v_t-(t-tt)\frac{dF_\theta}{dt} \]

这正是 MeanFlow 的 JVP target。所以 TwinFlow 的 residual target：

target = stopgrad(F_theta - clip(residual))
loss = mse(F_theta, target)

本质上是无显式 JVP 的 MeanFlow 训练。区别是它用模型自己的相邻时间预测构造 residual，因此必须控制自举噪声。

5. Fake Velocity 与 Adv 分支

需要特别区分：TwinFlow 的 fake/adv 分支不是 MeanFlow 的必要组成部分。

问题 澄清
为什么负时间代表 fake? 不是负号天然代表 fake。TwinFlow 人为把 real samples 送入正时间分支，把 model-generated fake samples 送入负时间分支。负时间只是复用 time embedding 的 domain label。
fake velocity 是什么? 它是当前模型生成分布 \(p_\theta\) 诱导出的 velocity field，而不是最终部署时要执行的动作速度。
它的作用是什么? 比较 fake_v - real_v，作为当前生成分布和真实数据分布之间的 correction signal，类似隐式 discriminator 或 score-difference。
FastWAM A2 是否采用? 第一版不采用。机器人动作 fake sample 可能包含任务语义错误，直接训练 fake branch 可能强化错误动作分布。
A2 的目标是先把 MeanFlow 主体训稳。fake/adv 可以作为 A3 的候选方向，但需要先设计动作级 fake sample 过滤、任务语义约束和 rollout failure 诊断。

6. 相关实现对照

工作 MeanFlow 实现方式 Anchor / Consistency 对 FastWAM 的启发
Official MeanFlow 显式 JVP，目标 \(v-(t-r)d u/dt\)。 大量 \(r=t\) diagonal/equal-time 样本；无 endpoint MSE。 理论基准清晰，但 action model 上 JVP 代价和工程风险较高。
SplitMeanFlow 区间拆分一致性，不直接算 JVP。 有 equal-time/boundary velocity anchor；主要 loss 是 interval splitting consistency。 说明 boundary/local anchor 与 MeanFlow 不矛盾，反而是稳定平均场的常见做法。
TwinFlow finite-difference residual，近似 MeanFlow JVP target。 混合 e2e、local/mul、any interval；有 residual clipping；另有 fake/adv 分支。 A2 借鉴 interval mixture 和 clipping，暂不借鉴 fake/adv。
MeanFlowSE 支持 finite-difference JVP，常用 centered FD 和 JVP clipping。 混合 CFM/equal-time loss 与 MeanFlow，带 warmup。 说明 clipped finite-difference + local flow anchor 是可靠的工程组合。

7. A2 方案定义

这里把 A2 拆成两层：A2 ablation ladder 用于归因，A2-core 是通过前两项 ablation 后的最小组合版。 Endpoint、hard-task upweight 和 rank 8 扩容不进入 A2-core，避免一次实验改动过多。

7.1 模型范围

模块 A2 策略 理由
Video expert 冻结 保留 release 视觉语义和 LIBERO/RoboTwin 泛化能力。
Proprio encoder 冻结 旧 action-only 经验显示 proprio 放开容易改变状态语义；A2 先避免引入新漂移源。

7.3 Interval Mixture

A2 不再只用单一 random interval。推荐初始采样：

Mode 概率 采样方式 作用
e2e 0.30 \(r=0,t=1\)，可选小 jitter：\(r\in[0,0.02]\)，\(t\in[0.98,1]\)。 直接训练 one-step rollout 所需的完整区间。
local 0.30 \(t\sim U(0.15,1)\)，\(\Delta=t-r\sim U(0.02,0.15)\)。 保持平均速度场在短区间接近 release flow matching。
random/any 0.40 随机 \(0\le r\lt t\le 1\)，最小 interval 仍保留。 学习真正的任意区间 MeanFlow，而不是只会 e2e 或 local。
Equal-time anchor 继续保留，但它是额外的 diagonal velocity anchor；local interval 则是非零长度的短区间平均速度训练，两者互补。

8. A2-core Loss

8.1 Clipped Residual MeanFlow

对任意 interval \(r\lt t\)，模型先预测：

\[ F_t = u_\theta(x_t,r,t) \]

沿瞬时速度 \(v_t\) 后退一个小步：

\[ x_{t-\Delta}=x_t-\Delta v_t \] \[ F_{t-\Delta}=u_\theta(x_{t-\Delta},r,t-\Delta) \]

构造积分残差：

\[ R=\frac{(t-r)F_t-(t-\Delta-r)F_{t-\Delta}}{\Delta}-v_t \]

裁剪 residual 后生成 stopgrad target：

\[ \tilde{R}=\mathrm{clip}(R,c) \] \[ y_{\mathrm{mf}}=\mathrm{sg}(F_t-\tilde{R}) \] \[ L_{\mathrm{mf}}=\|F_t-y_{\mathrm{mf}}\|^2 \]

固定点不变：当模型满足 MeanFlow 时 \(R=0\)，clipping 不起作用。clipping 只限制自举 target 在早期或 outlier batch 中的移动幅度。

8.2 Equal-Time Anchor

\[ L_{\mathrm{eq}}=\|u_\theta(x_s,s,s)-v_s\|^2 \]

这是对 release instantaneous velocity 的边界约束，不与 MeanFlow 矛盾。MeanFlow 在 \(r=t\) 的极限就应该退化到瞬时速度。

8.3 Endpoint Consistency

\[ \hat{a}_r=x_t-(t-r)u_\theta(x_t,r,t) \] \[ L_{\mathrm{endpoint}}=\|\hat{a}_r-a_0\|^2 \]

endpoint loss 不是标准 MeanFlow 必需项，但对 action one-step rollout 有现实意义。为了归因，A2-core 默认不加 endpoint；它只进入后续 A2-endpoint ablation。

8.4 总 Loss

\[ L_{\mathrm{A2}}= 1.0L_{\mathrm{mf}} +0.20L_{\mathrm{eq}} +0.00L_{\mathrm{endpoint}} \]

其中 \(L_{\mathrm{eq}}\) 可按概率采样，例如 equal_time_anchor_prob=0.25。若 A2-core 明确优于 A1，再单独测试 lambda_action_endpoint=0.02/0.05。

Full-state continuation 的 max_steps 使用全局 step：80000 表示从 70000 续训 10k，不是从头训练 80k。启动方式必须匹配 A1 70k 的 ZeRO/DeepSpeed state； 优先使用同等 4-GPU DeepSpeed/Accelerate 配置，避免再次触发单进程加载 ZeRO state 失败。

12. 训练与评测计划

12.1 数据

A2-core 继续使用 full LIBERO no-noops four-suite 数据，且暂不改变采样权重。困难任务上采样只进入 A2-data，否则无法判断收益来自 objective 还是数据分布。

数据策略 建议 理由
Full four-suite 保留 spatial/object/goal/libero_10 全部数据。 避免窄任务 finetune 破坏 release 泛化。
Hard task upweight A2-core 不启用；仅在 A2-data 中把 LIBERO-10 task 4/6/9 采样权重设为 2x 到 3x。 A1 弱项确实集中在这些任务，但先不把数据变量混入 objective 归因。
Control tasks 保留 task 2/3/5/7 等 A1 强项作为反退化检查。 避免只优化弱项导致整体退步。

12.2 阶段

阶段 规模 目标 通过条件
Preflight 0 step / checkpoint load 确认 release zero-update 行为、冻结参数、LoRA 初始化。 可训练参数列表正确；eval config 能加载。
Smoke 100 到 300 steps 检查 loss、residual norm、clip fraction、memory。 无 NaN；clip fraction 不长期接近 100%。
Pilot 10k steps 分别跑 A2-clip、A2-mix、A2-core，每个实验只改计划中指定变量。 hard task mini-eval 与 control tasks 同时不低于 A1 同等早期 checkpoint。
Main 30k / 70k steps 只对 pilot 胜出的最小变体做完整训练；endpoint、data、capacity 逐项追加。 每次只能新增一个变量，LIBERO-10 task 4/6/9 改善且强项不显著退步。

13. 监控指标

指标 含义 异常解释
loss/mf_e2e, loss/mf_local, loss/mf_random 不同 interval mode 的主 loss。 e2e 长期明显高，说明部署区间仍难；local 高，说明 release velocity anchor 被破坏。
meanflow/residual_norm finite-difference residual 大小。 长期上升或震荡说明自举 target 不稳。
meanflow/clip_fraction 被 residual clip 限制的 token 比例。 早期高可以接受；长期高于 30% 需要调小 lr、调大 interval 或放宽 clip。
loss/equal_time_velocity local/equal-time anchor 是否保持 release instantaneous velocity。 如果主 loss 降但 anchor 升，模型可能从 release velocity 漂走。
loss/action_endpoint one-step endpoint reconstruction 误差。 若 endpoint 降而 eval 退步，可能变成 shortcut overfit。
params/lora_norm, grads/action_head_norm LoRA 与 head 的有效更新幅度。 长期接近 0 是容量未用起来；突然暴涨是目标或 lr 不稳。

14. 风险与决策判据

风险 症状 处理
Endpoint loss 与 MeanFlow 冲突 endpoint error 降低，但 e2e eval 或 long-horizon task 退步。 把 lambda_action_endpoint 从 0.05 降到 0.02 或 0。
Residual target 自举噪声 clip fraction 长期很高，loss 波动大。 降低 lr；增大 local interval 下界；使用更保守的 residual clip。
LoRA 容量不足 loss 到平台但 hard task 不改善，LoRA norm 很小或饱和。 clean retrain 增加 rank 或 target modules；不要在 rank 不兼容 checkpoint 上直接 resume。
Hard task 过采样导致遗忘 task 4/6/9 提升，但 task 2/3/5/7 或 spatial/goal 退步。 降低 hard upweight，或改成 pilot 阶段 only 的 curriculum。
不以训练 loss 接近 0 作为成功标准。A2 的第一目标是 LIBERO-10 弱项提升，同时保持 A1 已经较强的 task 不退化。
