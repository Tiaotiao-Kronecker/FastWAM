本页按方案整理当前工作树中为了 one-step、shortcut、MeanFlow 及其评测诊断对 FastWAM 代码做过的改动。重点是“改了哪些源码文件、每个改动服务于哪个方案、训练/推理/评测路径如何接上”。

生成日期：2026-05-20。当前仓库处于 dirty worktree，本页反映的是本地代码状态，不是上游发布版本。

公共源码改动

这些改动不是某个单一方案的 loss，而是为了让 one-step/shortcut/MeanFlow 这些子类能在同一个 FastWAM 训练框架里挂载新模块、控制可训练参数、跑 bf16/JVP、保存或跳过训练状态。

模型训练钩子

文件改动服务对象
src/fastwam/models/wan22/fastwam.py 给基类新增 configure_trainable_parameters() 空钩子，子类可覆盖，用来冻结 video expert、打开 conditioner、关闭 proprio encoder 等。 one-step action、shortcut、MeanFlow、joint MeanFlow。
src/fastwam/trainer.py optimizer 收集参数时额外读取模型的 extra_trainable_parameters()，并按参数 id 去重。 动态挂到 action_expert / video_expert 上的 conditioner 模块。
src/fastwam/trainer.py 新增 save_training_state 配置项；保存 checkpoint 时可以只保存 weight checkpoint，跳过 DeepSpeed/Accelerate full state。 2k pilot 减少磁盘写入；joint pilot 可选择保留 full state 以便续训。

dtype / JVP 兼容

文件改动原因
src/fastwam/models/wan22/action_dit.py pre_dit 内把 action_tokens、timestep、context 转到对应 embedding weight 的 device/dtype。 修复 bf16 / fp32 混用时 Linear 输入 dtype 不一致，尤其是 JVP 和 meanflow smoke。
src/fastwam/models/wan22/wan_video_dit.py pre_dit 内把 video latent、timestep、context、可选 action 条件转到各自 embedding 的 device/dtype。 joint MeanFlow smoke 中 video 分支 bf16 dtype path 必需。
src/fastwam/models/wan22/wan_video_dit.py 新增 force_manual_attention()、is_manual_attention_forced()、_manual_scaled_dot_product_attention()，并让 flash_attention() 可绕开 PyTorch SDPA。 paper-JVP 路径里 forward-mode AD 不能稳定穿过 flash SDPA，JVP loss 需要 manual attention fallback。

One-Step Action Endpoint Fine-Tune

这是后续 shortcut 和 action-only MeanFlow 的基础子类：保留 FastWAM 的当前帧 video prefill 和 MoT action path，但训练时只用首帧 video token，直接在 action 端点学习。

文件具体改动
src/fastwam/models/wan22/fastwam_one_step_action.py 新增 FastWAMOneStepAction：
  • from_wan22_pretrained() 接收 one_step_action_timestep、loss_lambda_action_velocity、loss_lambda_action_endpoint、freeze_video_expert。
  • configure_trainable_parameters() 默认冻结 video_expert。
  • _build_one_step_action_timestep() 默认使用 action scheduler 最大 timestep，即 sigma=1 端点。
  • _masked_action_mse() 对 padded action token 做 mask 后求 MSE。
  • training_loss() 从 clean action 加噪到端点，预测 action velocity，并额外用 a_hat = noisy - sigma * pred_velocity 计算 endpoint loss。
src/fastwam/runtime.py 已有 create_fastwam_one_step_action() 工厂：解析 model/action scheduler/loss/one_step_action 配置，实例化 FastWAMOneStepAction。
configs/model/fastwam_one_step_action.yaml 新增模型配置，复用 Wan2.2-TI2V-5B video/action DiT 结构，设置 one_step_action.timestep=null 和 action loss 权重 0.5/0.5。
configs/task/robotwin_one_step_action_3cam_384_1e-4.yaml RoboTwin smoke task，10 steps，release checkpoint resume。

Shortcut 系列

B：工程版 Shortcut

文件具体改动
src/fastwam/models/wan22/fastwam_one_step_shortcut.py 新增/更新 FastWAMOneStepShortcut：
  • 继承 FastWAMOneStepAction，给 action expert 动态挂载 shortcut_step_embedding 和 shortcut_step_projection。
  • step-size conditioner 使用 sinusoidal embedding，把 shortcut_step_size 注入 action t_mod。
  • 训练时同一个 noisy action 同时预测大步 velocity、两个半步 velocity；loss 包括 action velocity、endpoint、shortcut consistency、half velocity。
  • 覆盖 _predict_action_noise_with_cache()，推理时在 action cache path 里也注入 shortcut step-size conditioner。
  • 当前工作树新增 configure_trainable_parameters() 与 extra_trainable_parameters()，确保动态 conditioner 进入 optimizer。
src/fastwam/runtime.py create_fastwam_one_step_shortcut() 解析 one_step_shortcut.start_timestep、step_size 和 shortcut loss 权重。
configs/model/fastwam_one_step_shortcut.yaml 模型配置：step_size=1.0，默认冻结 video expert，四项 loss 权重各 0.25。
configs/task/libero_one_step_shortcut_2cam224_1e-4.yaml LIBERO 2cam 2k pilot task，release checkpoint resume，save_training_state=false。

MeanFlow C / D0 / D1：Action-only 路线

这一组把 MeanFlow 接到 FastWAM 的 action 分支上。video expert 默认冻结，只编码首帧，action expert 学习区间平均速度 u_theta(z_t, r, t)。C/D0/D1 的区别主要是可训练参数范围。

核心模型类

文件具体改动
src/fastwam/models/wan22/fastwam_one_step_meanflow.py 新增 FastWAMOneStepMeanFlow：
  • 继承 FastWAMOneStepAction，保留 action-only fine-tune 和 video freeze 逻辑。
  • 动态给 action expert 挂载 meanflow_start_embedding / meanflow_start_projection，把区间起点 r 注入 action t_mod。
  • 支持 meanflow_trainable_scope：action、conditioner、conditioner_head。
  • 支持 meanflow_train_proprio_encoder，D0/D1 可关闭 proprio encoder 训练。
  • _sample_meanflow_sigma_pair() 支持固定端点或随机 r,t，支持 equal_time_prob。
  • 覆盖 _predict_action_noise_with_cache()，推理时用 r=start_timestep 的 conditioner 路径预测 action velocity。
src/fastwam/models/wan22/fastwam_one_step_meanflow.py paper-JVP objective：
  • _training_loss_paper_jvp() 构造 z_t=(1-t)a+t epsilon 和 v=epsilon-a。
  • 用 torch.func.jvp 计算 u 和方向导数 du/dt。
  • 训练目标为 u_target = v - (t-r) * du/dt，对 target stop-gradient。
  • JVP 内临时关闭 gradient checkpointing，并通过 force_manual_attention(True) 避开 flash SDPA。
src/fastwam/models/wan22/fastwam_one_step_meanflow.py finite-difference objective：
  • _training_loss_finite_difference() 随机采样 r<t，强制最小间隔为 derivative_epsilon。
  • 先 forward 得到 u_t，再构造 z_{t-eps}=z_t-eps*v，no-grad forward 得到 u_prev。
  • 用 (u_t.detach()-u_prev)/eps 近似 du/dt，再形成 MeanFlow target。
  • loss 可组合 lambda_meanflow_target、lambda_action_velocity、lambda_action_endpoint；当前配置只开 target loss。
src/fastwam/runtime.py 新增 create_fastwam_one_step_meanflow()，解析 objective、随机 r,t、equal-time 概率、可训练 scope、proprio 训练开关、loss 权重和 derivative epsilon。

配置区别

configs/task/libero_one_step_meanflow_2cam224_1e-4.yaml C-action-only：训练 action expert + MeanFlow conditioner + proprio encoder，video expert frozen，resume release checkpoint。

Joint Finite-Difference MeanFlow

该方案试图保持 FastWAMJoint 的 video/action 耦合结构，同时把 video 和 action 都改成 finite-difference MeanFlow 目标，用来验证 action-only MeanFlow 是否因为破坏 joint topology 而失败。

文件具体改动
src/fastwam/models/wan22/fastwam_joint_meanflow.py 新增 FastWAMJointMeanFlow：
  • 继承 FastWAMJoint，同时处理 noisy video latent 和 noisy action。
  • 给 video expert 和 action expert 分别安装 meanflow_start_embedding/projection，对两个分支都注入区间起点 r。
  • _sample_meanflow_sigma_pair() 对 video/action 共用随机 r,t。
  • _predict_joint_mean_velocity() 保持 MoT mixed attention，同时输出 video/action 平均速度。
  • 覆盖 _predict_joint_noise() 和 infer_action()，推理时也带 MeanFlow start conditioning。
  • configure_trainable_parameters() 支持 joint 或 conditioner scope。
src/fastwam/models/wan22/fastwam_joint_meanflow.py training loss：
  • 对 video/action 分别采样噪声并加到 sigma_end。
  • 保留 first-frame causal 约束：若有 first_frame_latents，noisy video 和 prev video 的第 0 帧强制替换为真实首帧。
  • 用一次当前 forward 和一次 no-grad prev forward 近似 video/action 的 du/dt。
  • video loss 使用原有 video mask 计算，action loss 使用 masked action MSE。
  • 额外记录 loss_video_endpoint_sanity 和 loss_action_endpoint_sanity，但不作为主 loss。
src/fastwam/runtime.py 新增 create_fastwam_joint_meanflow()，解析 joint_meanflow 配置、video/action MeanFlow loss 权重和可训练 scope。
configs/model/fastwam_joint_meanflow.yaml 模型配置：trainable_scope=joint，video/action MeanFlow loss 权重均为 1。
configs/task/libero_joint_meanflow_fd_2cam224_1e-4.yaml LIBERO joint finite-difference pilot，2k steps，save_every=500，save_training_state=true，resume release checkpoint。
当前结果：joint finite-difference 2k 权重 checkpoint 已保存，但 full state 保存因磁盘空间不足报错；用 step_002000.pt 评测 gap-probe 为 0/7。
