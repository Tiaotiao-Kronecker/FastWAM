FastWAM Joint Delta MeanFlow 总结

定义、实现方式、数据使用与当前实验结果，更新日期：2026-05-28

结论摘要

joint_delta 是当前 FastWAM MeanFlow 路线里最稳定的一版区间条件器。 它不是新的完整 video/action joint 模型，而是在 release action policy 上增加一个小型残差条件器， 让动作分支同时看到当前噪声时间 \(t\)、目标起点 \(r\) 和区间长度 \(t-r\)。

1. Flow Matching 原理与 FastWAM 实现

先从最基础的 Flow Matching 看。训练时我们从数据集中取一个干净动作 chunk \(a_0\)，再采一个高斯噪声 \(\epsilon_a\)。下文把归一化噪声强度直接记作 \(t\in[0,1]\)；代码里对应 timestep / num_train_timesteps。

\[ a_t=(1-t)a_0+t\epsilon_a \]

这里的 \(a_0\) 和 \(\epsilon_a\) 对一个训练样本来说是固定的，只有 \(t\) 在变。所以对 \(t\) 求导时：

\[ \frac{d a_t}{dt} = \frac{d}{dt}\left[(1-t)a_0+t\epsilon_a\right] = -a_0+\epsilon_a = \epsilon_a-a_0 = v_a \]

这就是为什么 \(\frac{d a_t}{dt}=v_a\)。它不是模型假设出来的，而是线性噪声路径本身的导数。 Flow Matching 要模型学习这个瞬时速度 \(v_a\)：

\[ v_a = \epsilon_a-a_0,\qquad \mathcal{L}_{\text{FM-action}} = w(t)\left\|v_{\theta,a}(a_t,t)-v_a\right\|_2^2 \]

对 video latent \(z_0\) 也是同样的过程：

\[ z_t=(1-t)z_0+t\epsilon_z,\qquad v_z=\epsilon_z-z_0 \] \[ \mathcal{L}_{\text{FM}} = \lambda_z w(t_z)\left\|v_{\theta,z}(z_t,t_z)-v_z\right\|_2^2 + \lambda_a w(t_a)\left\|v_{\theta,a}(a_t,t_a)-v_a\right\|_2^2 \]

FastWAM 的实现入口是 FastWAM.training_loss()。 WanContinuousFlowMatchScheduler 会先用 shift 函数采样训练时间：

\[ \sigma=\phi(u;s)=\frac{s u}{1+(s-1)u},\qquad u\sim\mathcal{U}(0,1),\quad s=5 \]

然后 add_noise() 构造 \((1-\sigma)x+\sigma\epsilon\)， training_target() 返回 \(\epsilon-x\)。推理时从噪声开始，用 Euler 更新从高噪声走到低噪声：

\[ 1=\sigma_0>\sigma_1>\cdots>\sigma_K=0,\qquad \Delta_i=\sigma_{i+1}-\sigma_i<0 \] \[ x_{\sigma_{i+1}} = x_{\sigma_i}+v_\theta(x_{\sigma_i},\sigma_i)\Delta_i \]

因为 \(\Delta_i<0\)，这一步实际是在沿预测速度的反方向从噪声回到数据。 release_1 就是 \(K=1\)，release_10 就是 \(K=10\)。

2. MeanFlow 原理

Flow Matching 学的是某个时间点的瞬时速度 \(v_\theta(a_t,t)\)。MeanFlow 换一个视角： 它希望模型直接输出从 \(t\) 到 \(r\) 这一整个区间的平均速度。下面为了简洁，把 action path 的 \(v_a\) 记为 \(v\)。

\[ u_\theta(a_t,r,t\mid I,q,p) \approx \frac{a_t-a_r}{t-r} \]

这个式子说明 \(u_\theta\) 的物理含义：如果平均速度正确，从 \(a_t\) 一步走长度 \(t-r\)， 应该到达 \(a_r\)：

\[ a_r\approx a_t-(t-r)u_\theta(a_t,r,t) \]

这个平均速度定义和导数项有直接关系。把理想平均速度写成 \((t-r)u(a_t,r,t)=a_t-a_r\)，固定 \(r\)，对终点时间 \(t\) 求导：

\[ \frac{d}{dt}\left[(t-r)u(a_t,r,t)\right] = \frac{d a_t}{dt} = v \] \[ u(a_t,r,t)+(t-r)\frac{d u(a_t,r,t)}{dt}=v \] \[ u(a_t,r,t)=v-(t-r)\frac{d u(a_t,r,t)}{dt} \]

最后一行就是 MeanFlow identity。训练时真实 \(v=\epsilon_a-a_0\) 是已知的，因为 noisy action 是我们用 \(a_0\) 和 \(\epsilon_a\) 构造出来的；模型则通过这个 identity 学会直接预测区间平均速度。

\[ u_{\text{target}} = v-(t-r)\frac{d u_\theta(a_t,r,t)}{dt} \] \[ \mathcal{L}_{\text{mf}} = \left\| u_\theta(a_t,r,t)-\operatorname{sg}(u_{\text{target}}) \right\|_2^2 \]

一步动作生成时取 \(r=0,t=1\)，从随机噪声动作 \(a_1\) 直接预测 \(a_0\)：

\[ \hat a_0=a_1-u_\theta(a_1,0,1) \]

3. MeanFlow 的 JVP 与 Finite Difference 版本

MeanFlow target 的难点是 \(\frac{d u_\theta}{dt}\)。这个导数是沿噪声路径的全导数： \(t\) 改变时，输入 action \(a_t\) 也必须一起改变。

\[ \frac{d u_\theta(a_t,r,t)}{dt} = \frac{\partial u_\theta}{\partial a_t}\frac{d a_t}{dt} + \frac{\partial u_\theta}{\partial t} = J_{a}u_\theta\cdot v+\partial_t u_\theta \]

JVP 版本

JVP 版本用自动微分直接算上面的全导数。实现上给 torch.func.jvp 的切向量是：

\[ \dot a_t=v,\qquad \dot t=1,\qquad \dot r=0 \]

它更贴近论文形式，但在当前 FastWAM/MoT 路径里显存和 checkpointing 约束更强，调试成本也更高。

Finite Difference 版本

当前 joint_delta 候选使用 finite difference。给定 \(\eta=0.05\)， 先沿同一条真实噪声路径退一步：

\[ a_{t-\eta} = (1-t+\eta)a_0+(t-\eta)\epsilon_a = a_t-\eta(\epsilon_a-a_0) = a_t-\eta v \]

然后做两次普通 forward，分别计算 \((a_t,r,t)\) 和 \((a_{t-\eta},r,t-\eta)\) 的输出：

\[ \frac{d u_\theta}{dt} \approx \frac{ u_\theta(a_t,r,t)- u_\theta(a_{t-\eta},r,t-\eta) }{\eta} \]

所以 finite difference 和 JVP 不是两个不同的 MeanFlow 目标；它们都在估计同一个 \(\frac{d u_\theta}{dt}\)。区别只是：JVP 用自动微分精确算当前模型的方向导数， finite difference 用两个相邻时间点的输出差做数值近似。

4. Joint Delta 条件器定义

Joint Delta 条件器

joint_delta 不是把新 token 拼到 action token 序列里，也不是改 MoT mixed attention。 它接在 action DiT 原始时间调制分支后面，给原始 t_mod 加一个由 MeanFlow 区间生成的 residual。

原始 action DiT 的时间调制

原始 action DiT 本身就有一条小网络把当前噪声时间 \(t\) 变成 time modulation。 这里 \(\phi(t)\) 是 sinusoidal timestep embedding，\(F\) 是 freq_dim， \(H\) 是 hidden_dim：

\[ \phi(t)\in\mathbb{R}^{F} \] \[ h_t=E_{\text{time}}(\phi(t)),\qquad E_{\text{time}}:\mathbb{R}^{F}\rightarrow\mathbb{R}^{H} \] \[ m(t)=P_{\text{time}}(h_t),\qquad P_{\text{time}}:\mathbb{R}^{H}\rightarrow\mathbb{R}^{6H} \] \[ m(t)\in\mathbb{R}^{6\times H} \] 
# src/fastwam/models/wan22/action_dit.py
self.time_embedding = nn.Sequential(
    nn.Linear(freq_dim, hidden_dim),
    nn.SiLU(),
    nn.Linear(hidden_dim, hidden_dim),
)
self.time_projection = nn.Sequential(
    nn.SiLU(),
    nn.Linear(hidden_dim, hidden_dim * 6),
)

t = self.time_embedding(sinusoidal_embedding_1d(self.freq_dim, timestep))
t_mod = self.time_projection(t).unflatten(1, (6, self.hidden_dim))

所以原始时间调制本身就是网络：先用 time_embedding 把 sinusoidal 时间编码映射到 hidden 维，再用 time_projection 输出 \(6H\) 个调制参数。当前 action expert 中 freq_dim=256、hidden_dim=1024，因此 t_mod 的形状是 \([B,6,1024]\)。

freq_dim 的作用是决定 sinusoidal timestep embedding 的维度。原始时间输入只是一个标量 \(t\)，直接送进 MLP 表达力较弱；先展开成 \(\phi(t)\in\mathbb{R}^{\text{freq\_dim}}\)，等价于给时间提供一组从低频到高频的正弦/余弦坐标， 让网络更容易区分不同噪声时间并学习时间相关的非线性变化。

\[ t\in\mathbb{R} \quad\rightarrow\quad \phi(t)\in\mathbb{R}^{\text{freq\_dim}} \quad\rightarrow\quad h_t\in\mathbb{R}^{H} \quad\rightarrow\quad t_{\text{mod}}\in\mathbb{R}^{6\times H} \] 
# action expert example
freq_dim = 256
hidden_dim = 1024

scalar timestep t
  -> sinusoidal_embedding_1d(freq_dim, t)  # [B, 256]
  -> time_embedding MLP                    # [B, 1024]
  -> time_projection                       # [B, 6 * 1024]
  -> unflatten                             # [B, 6, 1024]

这 6 组参数在每个 DiT block 中被拆成 self-attention 和 MLP 的 shift、scale、gate：

\[ m(t) \rightarrow [s_{\text{msa}},\ c_{\text{msa}},\ g_{\text{msa}},\ s_{\text{mlp}},\ c_{\text{mlp}},\ g_{\text{mlp}}] \] 
# src/fastwam/models/wan22/mot.py
shift_msa, scale_msa, gate_msa, shift_mlp, scale_mlp, gate_mlp = (
    block.modulation + t_mod
).chunk(6, dim=chunk_dim)

attn_input = modulate(block.norm1(x), shift_msa, scale_msa)
...
mlp_input = modulate(block.norm2(x), shift_mlp, scale_mlp)

因此 t_mod 不是普通输入 token；它是每层 transformer block 的 AdaLN/门控调制条件。 joint_delta 选择改这里，是因为可以在不改变 token 序列和 MoT attention 结构的前提下， 让 action 分支感知 MeanFlow 区间。

ActionDiT 与 VideoDiT 中的具体位置

时间调制不在 VAE、action encoder、text embedding 或 MoT attention mask 里。它是在每个 DiT expert 的 pre_dit() 阶段先算好，然后随 token 一起传入每一层 transformer block。

分支 生成位置 形状 使用位置
ActionDiT ActionDiT.pre_dit() [B,6,H] 每层 action transformer block 的 attention/MLP shift、scale、gate；action 输出 head 当前不再使用时间调制。
VideoDiT WanVideoDiT.pre_dit() [B,S,6,H]，其中 S=T_lat*H_lat*W_lat 是 video patch token 数 每层 video transformer block 的 token-wise shift、scale、gate；video 输出 head 还会再用 t 做一次 shift/scale。

ActionDiT 是 action 序列级调制：一个样本内所有 action token 共享同一个 \(m_a(t)\)。 它的时间调制生成后直接进入 action blocks：

\[ a_{\text{tok}}=\operatorname{Linear}_{a}(a_t),\qquad m_a(t)=P^a_{\text{time}}(E^a_{\text{time}}(\phi(t)))\in\mathbb{R}^{6\times H} \] \[ x^{\ell+1}_{a} = \operatorname{ActionBlock}_{\ell}\left(x^\ell_a,\ c,\ m_a(t)\right) \] 
# src/fastwam/models/wan22/action_dit.py
pre_state = self.pre_dit(action_tokens=action_tokens, timestep=timestep, ...)
x = pre_state["tokens"]
t_mod = pre_state["t_mod"]

for block in self.blocks:
    x = block(x, context, t_mod, freqs, context_mask=context_mask)

return self.head(x)

VideoDiT 在当前 FastWAM 配置中使用 token-wise 时间调制。因为 first-frame latent 是条件帧， token_timesteps[:, 0, :] 被置为 0；其余 video latent token 使用当前 noisy timestep。 这样第一帧 token 和待生成帧 token 在同一个 video expert 中使用不同时间条件：

\[ m_z(t_i)=P^z_{\text{time}}(E^z_{\text{time}}(\phi(t_i)))\in\mathbb{R}^{6\times H} \] \[ M_z=[m_z(t_1),\ldots,m_z(t_S)]\in\mathbb{R}^{S\times6\times H} \] 
# src/fastwam/models/wan22/wan_video_dit.py
token_timesteps = torch.ones((batch_size, x.shape[2], tokens_per_frame)) * timestep.view(batch_size, 1, 1)
token_timesteps[:, 0, :] = 0
token_timesteps = token_timesteps.reshape(batch_size, -1)

token_t_emb = sinusoidal_embedding_1d(self.freq_dim, token_timesteps.reshape(-1))
t = self.time_embedding(token_t_emb).reshape(batch_size, -1, self.hidden_dim)
t_mod = self.time_projection(t).unflatten(2, (6, self.hidden_dim))

VideoDiT block 使用 t_mod 的方式和 action block 一样，只是多了 token 维度； chunk_dim=2 表示在 [B,S,6,H] 的第 6 组调制维上切分： 

# src/fastwam/models/wan22/wan_video_dit.py
for block in self.blocks:
    x_tokens = block(
        x_tokens,
        context_emb,
        t_mod,
        freqs,
        context_mask=context_attn_mask,
        self_attn_mask=self_attn_mask,
    )

# video head also uses pre_state["t"]
x = self.head(x_tokens, pre_state["t"])

在 FastWAM 的 MoT 路径中，video 和 action 的 t_mod 分别通过 t_mod_all["video"] 与 t_mod_all["action"] 传给 MoT。MoT 每层先用各自 expert 的 t_mod 构造 Q/K/V 输入和 residual gate，再把 video/action 的 Q/K/V 拼起来做 mixed attention。 因此时间调制发生在 mixed attention 之前，并且每个 expert 保留自己的时间条件。 

# src/fastwam/models/wan22/mot.py
tokens_out = self.mot(
    embeds_all={"video": video_pre["tokens"], "action": action_pre["tokens"]},
    t_mod_all={"video": video_pre["t_mod"], "action": action_pre["t_mod"]},
    ...
)

# inside each MoT layer
q, k, v, residual_x, gate_msa, shift_mlp, scale_mlp, gate_mlp, _ = \
    self._build_expert_attention_io(..., t_mod=t_mod)

DiTBlock 里的 LayerNorm 与时间调制

每层 DiTBlock 里有三处 LayerNorm：norm1、norm2 和 norm3。 其中 norm1/norm2 关闭了 LayerNorm 自己的 affine 参数， 因为它们后面会接时间调制生成的 shift/scale；norm3 保留 affine 参数， 因为 cross-attention 分支没有额外的 timestep shift/scale。 

# src/fastwam/models/wan22/wan_video_dit.py
self.norm1 = nn.LayerNorm(hidden_dim, eps=eps, elementwise_affine=False)
self.norm2 = nn.LayerNorm(hidden_dim, eps=eps, elementwise_affine=False)
self.norm3 = nn.LayerNorm(hidden_dim, eps=eps)

LN(no affine) 只做标准化，没有可学习的 \(\gamma,\beta\)：

\[ \operatorname{LN}_{\text{no-affine}}(x) = \frac{x-\mu}{\sqrt{\sigma^2+\epsilon}} \]

然后由时间调制提供 scale 和 shift：

\[ \operatorname{modulate}(\operatorname{LN}(x),s,c) = \operatorname{LN}(x)\cdot(1+c)+s \] 
# self-attention input
input_x = modulate(self.norm1(x), shift_msa, scale_msa)

# MLP input
input_x = modulate(self.norm2(x), shift_mlp, scale_mlp)

LN(affine) 则保留可学习的 \(\gamma,\beta\)：

\[ \operatorname{LN}_{\text{affine}}(x) = \gamma\frac{x-\mu}{\sqrt{\sigma^2+\epsilon}}+\beta \] 
# cross-attention input
x = x + self.cross_attn(self.norm3(x), context, ctx_mask=context_mask)

因此，timestep 生成的 t_mod 主要控制 self-attention 和 MLP 两条主干分支： shift_msa/scale_msa/gate_msa 控制 self-attention， shift_mlp/scale_mlp/gate_mlp 控制 MLP；cross-attention 分支是普通 affine LayerNorm 加 residual。

早期 additive_start 条件器

早期 additive_start 只把起点 \(r\) 编码成 residual，叠加到原始 \(m(t)\) 上：

\[ \tilde m(t,r) = m(t) + \Delta m_r,\qquad \Delta m_r = g_r(\phi(r)) \] 
# src/fastwam/models/wan22/fastwam_one_step_meanflow.py
start_emb = sinusoidal_embedding_1d(self.action_expert.freq_dim, start_position)
start_hidden = self.action_expert.meanflow_start_embedding(start_emb)
start_mod = self.action_expert.meanflow_start_projection(start_hidden).unflatten(
    1, (6, self.action_expert.hidden_dim)
)
action_pre["t_mod"] = action_pre["t_mod"] + start_mod

这个版本的问题是新增 residual 只依赖 \(r\)，没有显式看到当前时间 \(t\) 和区间长度 \(t-r\)。

当前 joint_delta 条件器网络

当前 joint_delta 把当前时间、起点时间和区间长度联合编码。它仍然输出一个 residual， 但 residual 来自 \(f(t,r,t-r)\)，而不是只来自 \(f(r)\)：

\[ e_{\text{jd}}(t,r) = \left[\phi(t),\ \phi(r),\ \phi(t-r)\right] \] \[ \Delta m_{\text{jd}} = P_{\text{jd}}\left(E_{\text{jd}}(e_{\text{jd}})\right) \in \mathbb{R}^{6\times H} \] \[ \tilde m_{\text{jd}}(t,r) = m(t) + \Delta m_{\text{jd}}(t,r) \]

对应网络结构是两段 MLP。第一段把拼接后的区间 embedding 从 \(3F\) 映射到 \(H\)， 第二段把 hidden 表示投影成 \(6H\) 个调制 residual： 

# src/fastwam/models/wan22/fastwam_one_step_meanflow.py
self.action_expert.meanflow_joint_time_embedding = nn.Sequential(
    nn.Linear(freq_dim * 3, hidden_dim),
    nn.SiLU(),
    nn.Linear(hidden_dim, hidden_dim),
)
self.action_expert.meanflow_joint_time_projection = nn.Sequential(
    nn.SiLU(),
    nn.Linear(hidden_dim, hidden_dim * 6),
)
# src/fastwam/models/wan22/fastwam_one_step_meanflow.py
action_emb = sinusoidal_embedding_1d(self.action_expert.freq_dim, action_position)
start_emb = sinusoidal_embedding_1d(self.action_expert.freq_dim, start_position)
interval_emb = sinusoidal_embedding_1d(self.action_expert.freq_dim, interval_position)
joint_emb = torch.cat([action_emb, start_emb, interval_emb], dim=1)

joint_hidden = self.action_expert.meanflow_joint_time_embedding(joint_emb)
joint_mod = self.action_expert.meanflow_joint_time_projection(joint_hidden).unflatten(
    1, (6, self.action_expert.hidden_dim)
)
action_pre["t_mod"] = action_pre["t_mod"] + joint_mod

这里 \(\phi(\cdot)\) 是 sinusoidal timestep embedding。当前 action expert 中 freq_dim=256，因此拼接后的 joint_emb 维度是 \(256\times3=768\)； hidden_dim=1024，输出 residual 形状是 \([B,6,1024]\)。 最后一层零初始化，所以训练起点严格接近 release policy：

\[ \Delta m_{\text{jd}}(t,r)=0 \quad\Rightarrow\quad \tilde m_{\text{jd}}(t,r)=m(t) \]
术语澄清： joint_delta 的 "joint" 指联合编码 \(t,r,t-r\) 这个 MeanFlow 区间， 不是指训练时让 action token 访问完整未来 video token。当前候选仍是 release-style first-frame action policy。

save_every: 500

推理路径

评测时仍走 release action policy 风格：输入当前图像，编码 first-frame latent，prefill video KV cache， 然后对 action noise 做动作 denoise。对于当前 one-step readout：

\[ r=0,\quad t=1,\quad a_0 = a_1 - u_\theta(a_1,0,1) \]

因此 NUM_INFERENCE_STEPS=1 时它是干净的一步生成。若设置多步推理， 当前实现每一步仍固定 \(r=0\)，不能解释为严格的局部区间 solver。

样本构造与归一化

RobotVideoDataset 读取 LeRobot 数据后，FastWAMProcessor 负责把图像、动作、 state/proprio 和任务文本整理成训练样本。一个 batch 进入模型前的主要字段是： video、action、proprio、context、context_mask， 以及 padding mask。

\[ \text{video}\in\mathbb{R}^{B\times 3\times T\times H\times W},\quad \text{action}\in\mathbb{R}^{B\times 32\times d_a},\quad \text{proprio}\in\mathbb{R}^{B\times 33\times d_p} \]

action/state 的归一化由 LinearNormalizer 完成。LIBERO 当前使用 min/max，RoboTwin 当前使用 z-score：

\[ \text{min/max:}\quad \tilde{x} = 2\frac{x-x_{\min}}{x_{\max}-x_{\min}}-1 \] \[ \text{z-score:}\quad \tilde{x} = \frac{x-\mu}{\sigma+10^{-8}} \]

归一化后的值会被 clamp 到 \([-5,5]\)。多字段 action/state 通过 ConcatLeftAlign 左对齐拼接；LIBERO 的 delta_action_dim_mask 指定 eef pose 6 维是 delta action，gripper 维不是 delta action。
